# 어드모니션

어드모니션은 문서에서 중요한 정보, 경고, 팁, 메모를 강조하는 데 도움이 되는 특별한 콜아웃 박스입니다. BaiDocs는 사용자 정의 가능한 스타일링과 함께 풍부한 내장 어드모니션 유형 세트를 제공합니다.

## 기본 어드모니션

```mdx

:::

:::note 사용자 정의 제목
메모에 사용자 정의 제목을 제공할 수도 있습니다.
:::
```

## 사용자 정의 어드모니션 유형

### 사용자 정의 유형 생성

특정 사용 사례를 위한 사용자 정의 어드모니션 유형을 생성할 수 있습니다:

```mdx
:::api API 참조
**엔드포인트:** `GET /api/users`

**매개변수:**
- `limit` (선택사항): 반환할 최대 사용자 수 (기본값: 10)
- `offset` (선택사항): 건너뛸 사용자 수 (기본값: 0)

**응답:**
```json
{
  "users": [...],
  "total": 150,
  "limit": 10,
  "offset": 0
}
```
:::

:::example 코드 예제
사용자 인증을 구현하는 방법입니다:

```javascript
import { authenticateUser } from './auth';

async function loginUser(email, password) {
  try {
    const user = await authenticateUser(email, password);
    return { success: true, user };
  } catch (error) {
    return { success: false, error: error.message };
  }
}
```
:::
```

### 테마 어드모니션

브랜드나 특정 테마에 맞는 어드모니션을 생성할 수 있습니다:

```mdx
:::theme-primary 브랜드 메시지
이 어드모니션은 기본 브랜드 색상과 스타일링을 사용합니다.
:::

:::theme-secondary 추가 정보
덜 눈에 띄는 정보에 보조 브랜드 색상을 사용합니다.
:::

:::theme-accent 강조된 콘텐츠
특별한 주의가 필요한 콘텐츠에 액센트 색상을 사용합니다.
:::
```

## 모범 사례

### 각 유형을 언제 사용할지

| 유형 | 사용 사례 | 예제 |
|------|----------|---------|
| **Note** | 일반 정보, 맥락 | "이 기능에는 인증이 필요합니다" |
| **Tip** | 모범 사례, 제안 | "팁: 설명적인 커밋 메시지를 사용하세요" |
| **Info** | 중립적 정보 | "API 속도 제한은 시간당 1000 요청입니다" |
| **Warning** | 잠재적 문제 | "경고: 이는 기존 파일을 덮어씁니다" |
| **Danger** | 중요한 경고 | "위험: 이 작업은 데이터를 영구적으로 삭제합니다" |
| **Success** | 긍정적 결과 | "성공: 설치가 완료되었습니다" |

### 콘텐츠 가이드라인

**간결하게**:
```mdx
<!-- 좋음 -->
:::warning
계속하기 전에 데이터를 백업하세요.
:::

<!-- 피할 것 -->
:::warning
무언가 잘못될 가능성이 있기 때문에 이 과정을 계속하기 전에 모든 중요한 데이터 파일의 백업을 생성하는 것을 고려해야 한다고 강력히 권장됩니다.
:::
```

**능동태 사용**:
```mdx
<!-- 좋음 -->
:::tip
프로덕션에 배포하기 전에 테스트를 실행하세요.
:::

<!-- 피할 것 -->
:::tip
프로덕션 배포가 수행되기 전에 테스트가 실행되어야 합니다.
:::
```

**맥락 제공**:
```mdx
<!-- 좋음 -->
:::warning 파일 권한
웹 서버가 uploads 디렉토리에 쓰기 권한을 갖도록 하세요. Unix 시스템에서는 `chmod 755 uploads/`를 사용하세요.
:::

<!-- 피할 것 -->
:::warning
권한을 확인하세요.
:::
```

### 사용자 정의 아이콘

어드모니션 아이콘을 사용자 정의할 수 있습니다:

```mdx
:::note 📝 문서 메모
문서 관련 메모를 위한 사용자 정의 아이콘입니다.
:::

:::tip 💡 빠른 팁
밝은 아이디어를 위한 전구 아이콘입니다.
:::

:::warning ⚠️ 중요한 경고
주의를 끄는 경고를 위한 경고 삼각형입니다.
:::
```

## 접근성

### 스크린 리더 지원

어드모니션은 접근성을 염두에 두고 설계되었습니다:

```mdx
:::warning 스크린 리더 지원
이 경고는 적절한 ARIA 레이블과 역할을 가지고 스크린 리더에 알려집니다.
:::
```

### 키보드 내비게이션

사용자는 키보드를 사용하여 어드모니션을 탐색할 수 있습니다:
- **Tab**: 다음 인터랙티브 요소로 이동
- **Enter/Space**: 세부사항 어드모니션 펼치기/접기
- **Escape**: 펼쳐진 세부사항 닫기

### 색상 대비

모든 어드모니션 유형은 WCAG 2.1 AA 색상 대비 요구사항을 충족합니다:
- 텍스트 대비 비율: 최소 4.5:1
- 아이콘 대비 비율: 최소 3:1
- 포커스 표시기: 높은 대비 테두리

## 고급 예제

### 다단계 프로세스

적절한 어드모니션과 함께 복잡한 프로세스를 명확한 단계로 나눕니다:

```mdx
# 데이터베이스 마이그레이션 가이드

:::info 마이그레이션 개요
이 프로세스는 데이터베이스를 버전 1.0에서 2.0으로 마이그레이션합니다. 예상 다운타임: 15-30분.
:::

## 1단계: 준비

:::warning 백업 필요
시작하기 전에 전체 데이터베이스 백업을 생성하세요:
```bash
pg_dump myapp_production > backup_$(date +%Y%m%d).sql
```
:::

## 2단계: 마이그레이션 실행

:::danger 프로덕션 영향
이 단계는 데이터베이스를 잠그고 쓰기를 방지합니다:
```bash
npm run migrate:up
```
:::

## 3단계: 검증

:::success 검증
마이그레이션이 성공했는지 확인하세요:
```bash
npm run migrate:status
```
:::

:::tip 마이그레이션 후
안정성을 보장하기 위해 마이그레이션 후 30분 동안 애플리케이션 로그를 모니터링하세요.
:::
```

### API 문서

API 문서를 향상시키기 위해 어드모니션을 사용합니다:

```mdx
# 사용자 인증 API

## POST /auth/login

사용자를 인증하고 액세스 토큰을 반환합니다.

:::api 요청 형식
```json
{
  "email": "user@example.com",
  "password": "securepassword"
}
```
:::

:::success 성공 응답
```json
{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 123,
    "email": "user@example.com",
    "name": "홍길동"
  }
}
```
:::

:::warning 오류 응답
```json
{
  "success": false,
  "error": "잘못된 자격 증명"
}
```
:::

:::note 속도 제한
이 엔드포인트는 IP 주소당 분당 5회 시도로 속도가 제한됩니다.
:::
```
