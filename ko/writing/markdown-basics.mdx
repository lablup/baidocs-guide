# 마크다운 기초

BaiDocs는 MDX (Markdown + JSX)를 주요 콘텐츠 형식으로 사용합니다. 이는 표준 마크다운의 모든 기능과 React 구성 요소 및 사용자 정의 스타일링을 포함할 수 있는 능력을 제공합니다.

## 표준 마크다운 문법

모든 표준 마크다운 문법이 BaiDocs에서 완전히 지원됩니다:

### 헤더

헤더에는 `#`를 사용합니다. BaiDocs는 헤더에서 자동으로 내비게이션과 목차를 생성합니다:

```markdown
# H1 헤더 (페이지 제목)
## H2 헤더 (주요 섹션)
### H3 헤더 (하위 섹션)
#### H4 헤더 (하위 하위 섹션)
##### H5 헤더 (작은 섹션)
###### H6 헤더 (가장 작은 헤더)
```

**결과:**
# H1 헤더 (페이지 제목)
## H2 헤더 (주요 섹션)
### H3 헤더 (하위 섹션)
#### H4 헤더 (하위 하위 섹션)
##### H5 헤더 (작은 섹션)
###### H6 헤더 (가장 작은 헤더)

### 텍스트 서식

간단한 문법으로 텍스트 서식을 지정할 수 있습니다:

```markdown
**굵은 텍스트** 또는 __굵은 텍스트__
*기울임 텍스트* 또는 _기울임 텍스트_
***굵고 기울임*** 또는 ___굵고 기울임___
~~취소선 텍스트~~
`인라인 코드`
```

**결과:**
**굵은 텍스트** 또는 __굵은 텍스트__
*기울임 텍스트* 또는 _기울임 텍스트_
***굵고 기울임*** 또는 ___굵고 기울임___
~~취소선 텍스트~~
`인라인 코드`

### 목록

순서가 있는 목록과 순서가 없는 목록을 생성할 수 있습니다:

```markdown
## 순서 없는 목록
- 항목 1
- 항목 2
  - 중첩된 항목 2.1
  - 중첩된 항목 2.2
- 항목 3

## 순서 있는 목록
1. 첫 번째 항목
2. 두 번째 항목
   1. 중첩된 항목 2.1
   2. 중첩된 항목 2.2
3. 세 번째 항목

## 혼합 목록
1. 순서 있는 항목
   - 순서 없는 중첩 항목
   - 또 다른 순서 없는 항목
2. 또 다른 순서 있는 항목
```

**결과:**

## 순서 없는 목록
- 항목 1
- 항목 2
  - 중첩된 항목 2.1
  - 중첩된 항목 2.2
- 항목 3

## 순서 있는 목록
1. 첫 번째 항목
2. 두 번째 항목
   1. 중첩된 항목 2.1
   2. 중첩된 항목 2.2
3. 세 번째 항목

### 링크

다양한 유형의 링크를 생성할 수 있습니다:

```markdown
[링크 텍스트](https://example.com)
[제목이 있는 링크](https://example.com "이것은 제목입니다")
[내부 링크](../getting-started/overview.mdx)
[앵커 링크](#headers)

<!-- 참조 스타일 링크 -->
[참조 링크][1]
[1]: https://example.com "참조 제목"

<!-- 자동 링크 -->
https://example.com
<user@example.com>
```

**결과:**
[링크 텍스트](https://example.com)
[제목이 있는 링크](https://example.com "이것은 제목입니다")
[내부 링크](../getting-started/overview.mdx)
[앵커 링크](#headers)

### 이미지

다양한 옵션으로 이미지를 포함할 수 있습니다:

```markdown
![대체 텍스트](../_assets/example.png)
![제목이 있는 대체 텍스트](../_assets/example.png "이미지 제목")

<!-- 참조 스타일 이미지 -->
![대체 텍스트][image1]
[image1]: ../_assets/example.png "참조 이미지"

<!-- 크기 지정 (HTML 스타일) -->
<img src="../_assets/example.png" alt="대체 텍스트" width="300" height="200" />
```

### 코드 블록

BaiDocs는 코드 블록에 대한 향상된 구문 강조를 제공합니다:

````markdown
```javascript
function greetUser(name) {
  console.log(`Hello, ${name}!`);
  return `Welcome to BaiDocs, ${name}`;
}

const user = "Developer";
greetUser(user);
```

```python
def greet_user(name):
    print(f"Hello, {name}!")
    return f"Welcome to BaiDocs, {name}"

user = "Developer"
greet_user(user)
```

```yaml
# 설정 예제
id: my-book
title: My Documentation
languages:
  - en
  - ko
navigation:
  en:
    - title: "Home"
      path: "index.mdx"
```
````

**결과:**

```javascript
function greetUser(name) {

| 더 많은 텍스트 | 더 많은 텍스트 | 더 많은 텍스트 |
```

## BaiDocs 향상 마크다운

BaiDocs는 추가 기능으로 표준 마크다운을 확장합니다:

### Frontmatter

YAML frontmatter를 사용하여 페이지에 메타데이터를 추가할 수 있습니다:

```yaml
---
title: 사용자 정의 페이지 제목
description: SEO를 위한 페이지 설명
author: 작성자 이름
date: 2024-01-26
tags:
  - documentation
  - guide
featured: true
---

# 페이지 콘텐츠

페이지 콘텐츠가 여기서 시작됩니다.
```

### 작업 목록

인터랙티브 체크박스를 생성할 수 있습니다:

```markdown
## 할 일 목록

- [x] 문서 작성
- [x] 예제 추가
- [ ] 콘텐츠 검토
- [ ] 프로덕션에 게시

## 쇼핑 목록

- [x] ~~식료품 구입~~ (완료)
- [ ] 드라이클리닝 픽업
- [ ] 치과 예약
```

**결과:**

## 할 일 목록

- [x] 문서 작성
- [x] 예제 추가
- [ ] 콘텐츠 검토
- [ ] 프로덕션에 게시

### 각주

추가 정보를 위한 각주를 추가할 수 있습니다:

```markdown
이것은 각주가 있는 문장입니다[^1].

여기에는 더 긴 각주가 있는 또 다른 문장이 있습니다[^longnote].

[^1]: 이것은 간단한 각주입니다.

[^longnote]: 이것은 여러 문단이 있는 더 긴 각주입니다.

    각주에는 여러 문단을 포함할 수 있습니다.

    - 그리고 목록도!
    - 이런 것들 말입니다.
```

### 정의 목록

용어집을 위한 정의 목록을 생성할 수 있습니다:

```markdown
용어 1
: 용어 1에 대한 정의

용어 2
: 용어 2에 대한 정의
: 용어 2에 대한 또 다른 정의

*강조가 있는* 용어
: 정의는 **다른** `마크다운` 요소를 포함할 수 있습니다.
```

## 작성 모범 사례

### 문서 구조

**논리적 계층 구조 사용**:
```markdown
# 페이지 제목 (H1) - 페이지당 하나만
## 주요 섹션 (H2)
### 하위 섹션 (H3)
#### 하위 하위 섹션 (H4)
```

**레벨 건너뛰기 방지**:
```markdown
# 좋은 구조
## 섹션 1
### 하위 섹션 1.1
### 하위 섹션 1.2
## 섹션 2

# 나쁜 구조 (H2를 건너뜀)
# 제목
### 하위 섹션 (H1에서 H3로 점프)
```

### 콘텐츠 가이드라인

**설명적으로 작성**:
```markdown
<!-- 좋음 -->
## 개발 환경 설치
### macOS 사용자를 위한 전제 조건

<!-- 피할 것 -->
## 설정
### Mac 관련 사항
```

**능동태 사용**:
```markdown
<!-- 좋음 -->
변경사항을 저장하려면 "저장" 버튼을 클릭합니다.

<!-- 피할 것 -->
"저장" 버튼을 클릭하여 변경사항을 저장할 수 있습니다.
```

## 일반적인 마크다운 함정

### 간격 문제

**문제**: 헤더 주변의 일관되지 않은 간격
```markdown
<!-- 잘못된 예 -->
## 헤더
헤더 바로 다음의 콘텐츠

## 또 다른 헤더
```

**해결책**: 헤더 주변에 빈 줄 추가
```markdown
<!-- 올바른 예 -->
## 헤더

적절한 간격이 있는 콘텐츠.

## 또 다른 헤더

적절한 간격이 있는 더 많은 콘텐츠.
```

### 목록 형식

**문제**: 잘못된 목록 들여쓰기
```markdown
<!-- 잘못된 예 -->
- 항목 1
  - 중첩된 항목 (잘못된 들여쓰기)
- 항목 2
```

**해결책**: 일관된 2칸 들여쓰기 사용
```markdown
<!-- 올바른 예 -->
- 항목 1
  - 중첩된 항목 (올바른 들여쓰기)
- 항목 2
```

### 링크 참조

**문제**: 깨진 상대 링크
```markdown
<!-- 잘못된 예 -->
[링크](../wrong/path.mdx)
```

**해결책**: 파일 경로 확인
```markdown
<!-- 올바른 예 -->
[링크](../correct/path.mdx)
```
