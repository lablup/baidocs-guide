# 마크다운 기초

BaiDocs는 MDX (Markdown + JSX)를 주요 콘텐츠 형식으로 사용합니다. 이는 표준 마크다운의 모든 기능과 React 컴포넌트 및 사용자 정의 스타일링을 임베드할 수 있는 능력을 제공합니다.

## 표준 마크다운 구문

모든 표준 마크다운 구문이 BaiDocs에서 완전히 지원됩니다:

### 헤더

헤더에는 `#`을 사용하세요. BaiDocs는 헤더에서 네비게이션과 목차를 자동으로 생성합니다:

```markdown
# H1 헤더 (페이지 제목)
## H2 헤더 (주요 섹션)
### H3 헤더 (하위 섹션)
#### H4 헤더 (하위 하위 섹션)
##### H5 헤더 (소 섹션)
###### H6 헤더 (가장 작은 헤더)
```

**결과:**
# H1 헤더 (페이지 제목)
## H2 헤더 (주요 섹션)
### H3 헤더 (하위 섹션)
#### H4 헤더 (하위 하위 섹션)
##### H5 헤더 (소 섹션)
###### H6 헤더 (가장 작은 헤더)

### 텍스트 서식

간단한 구문으로 텍스트를 서식화하세요:

```markdown
**굵은 텍스트** 또는 __굵은 텍스트__
*이탤릭 텍스트* 또는 _이탤릭 텍스트_
***굵고 이탤릭*** 또는 ___굵고 이탤릭___
~~취소선 텍스트~~
`인라인 코드`
```

**결과:**
**굵은 텍스트** 또는 __굵은 텍스트__
*이탤릭 텍스트* 또는 _이탤릭 텍스트_
***굵고 이탤릭*** 또는 ___굵고 이탤릭___
~~취소선 텍스트~~
`인라인 코드`

### 목록

순서가 있는 목록과 없는 목록을 만드세요:

```markdown
## 순서 없는 목록
- 항목 1
- 항목 2
  - 중첩된 항목 2.1
  - 중첩된 항목 2.2
- 항목 3

## 순서 있는 목록
1. 첫 번째 항목
2. 두 번째 항목
   1. 중첩된 항목 2.1
   2. 중첩된 항목 2.2
3. 세 번째 항목

## 혼합 목록
1. 순서 있는 항목
   - 순서 없는 중첩 항목
   - 또 다른 순서 없는 항목
2. 또 다른 순서 있는 항목
```

**결과:**

## 순서 없는 목록
- 항목 1
- 항목 2
  - 중첩된 항목 2.1
  - 중첩된 항목 2.2
- 항목 3

## 순서 있는 목록
1. 첫 번째 항목
2. 두 번째 항목
   1. 중첩된 항목 2.1
   2. 중첩된 항목 2.2
3. 세 번째 항목

### 링크

다양한 유형의 링크를 만드세요:

```markdown
[링크 텍스트](https://example.com)
[제목이 있는 링크](https://example.com "이것은 제목입니다")
[내부 링크](../getting-started/overview.mdx)
[앵커 링크](#헤더)

<!-- 참조 스타일 링크 -->
[참조 링크][1]
[1]: https://example.com "참조 제목"

<!-- 자동 링크 -->
https://example.com
<user@example.com>
```

**결과:**
[링크 텍스트](https://example.com)
[제목이 있는 링크](https://example.com "이것은 제목입니다")
[내부 링크](../getting-started/overview.mdx)
[앵커 링크](#헤더)

### 이미지

다양한 옵션으로 이미지를 임베드하세요:

```markdown

![제목이 있는 대체 텍스트](../_assets/example.png "이미지 제목")

<!-- 참조 스타일 이미지 -->
![대체 텍스트][image1]
[image1]: ../_assets/example.png "참조 이미지"

<!-- 크기 지정 (HTML 스타일) -->

```

### 코드 블록

BaiDocs는 코드 블록에 대한 향상된 구문 하이라이팅을 제공합니다:

````markdown
```javascript
function greetUser(name) {
  console.log(`안녕하세요, ${name}님!`);
  return `BaiDocs에 오신 것을 환영합니다, ${name}님`;
}

const user = "개발자";
greetUser(user);
```

```python
def greet_user(name):
    print(f"안녕하세요, {name}님!")
    return f"BaiDocs에 오신 것을 환영합니다, {name}님"

user = "개발자"
greet_user(user)
```

```yaml
# 설정 예제

title: 내 문서
languages:
  - en
  - ko

  en:
    - title: "홈"
      path: "index.mdx"
```
````

**결과:**

## BaiDocs 향상 마크다운

BaiDocs는 추가 기능으로 표준 마크다운을 확장합니다:

### 프론트매터

YAML 프론트매터를 사용하여 페이지에 메타데이터를 추가하세요:

```yaml
---
title: 사용자 정의 페이지 제목
description: SEO를 위한 페이지 설명
author: 작성자 이름
date: 2024-01-26
tags:
  - 문서
  - 가이드
featured: true
---

# 페이지 콘텐츠

페이지 콘텐츠가 여기서 시작됩니다.
```

### 작업 목록

상호작용 체크박스를 만드세요:

```markdown
## 할 일 목록

- [x] 문서 작성
- [x] 예제 추가
- [ ] 콘텐츠 검토
- [ ] 프로덕션에 게시

## 쇼핑 목록

- [x] ~~장보기~~ (완료)
- [ ] 세탁물 찾기
- [ ] 치과 예약
```

**결과:**

## 할 일 목록

- [x] 문서 작성
- [x] 예제 추가
- [ ] 콘텐츠 검토
- [ ] 프로덕션에 게시

### 각주

추가 정보를 위한 각주를 추가하세요:

```markdown
이것은 각주가 있는 문장입니다[^1].

더 긴 각주가 있는 또 다른 문장입니다[^longnote].

[^1]: 이것은 간단한 각주입니다.

[^longnote]: 이것은 여러 단락이 있는 더 긴 각주입니다.

    각주에 여러 단락을 포함할 수 있습니다.

    - 심지어 목록도!
    - 이런 식으로.
```

### 정의 목록

용어집을 위한 정의 목록을 만드세요:

```markdown
용어 1
: 용어 1에 대한 정의

용어 2
: 용어 2에 대한 정의
: 용어 2에 대한 또 다른 정의

*강조*가 있는 용어
: 정의에는 **다른** `마크다운` 요소가 포함될 수 있습니다.
```

## 작성 모범 사례

### 문서 구조

**논리적 계층 사용**:
```markdown
# 페이지 제목 (H1) - 페이지당 하나만
## 주요 섹션 (H2)
### 하위 섹션 (H3)
#### 하위 하위 섹션 (H4)
```

**수준 건너뛰기 금지**:
```markdown
# 좋은 구조
## 섹션 1
### 하위 섹션 1.1
### 하위 섹션 1.2
## 섹션 2

# 나쁜 구조 (H2 건너뛰기)
# 제목
### 하위 섹션 (H1에서 H3로 점프)
```

### 콘텐츠 가이드라인

**설명적으로 작성**:
```markdown
<!-- 좋음 -->
## 개발 환경 설치
### macOS 사용자를 위한 전제 조건

<!-- 피할 것 -->
## 설정
### Mac 관련
```

**능동태 사용**:
```markdown
<!-- 좋음 -->
"저장" 버튼을 클릭하여 변경사항을 저장하세요.

<!-- 피할 것 -->
"저장" 버튼을 클릭하면 변경사항이 저장됩니다.
```

**맥락 제공**:

  timeout: 5000

## 일반적인 마크다운 함정

### 공백 문제

**문제**: 헤더 주변의 일관되지 않은 공백
```markdown
<!-- 잘못됨 -->
## 헤더
헤더 바로 다음의 콘텐츠

## 다른 헤더
```

**해결책**: 헤더 주변에 빈 줄 추가
```markdown
<!-- 올바름 -->
## 헤더

적절한 공백이 있는 콘텐츠.

## 다른 헤더

적절한 공백이 있는 더 많은 콘텐츠.
```

### 목록 서식

**문제**: 잘못된 목록 들여쓰기
```markdown
<!-- 잘못됨 -->
- 항목 1
  - 중첩된 항목 (잘못된 들여쓰기)
- 항목 2
```

**해결책**: 일관된 2칸 들여쓰기 사용
```markdown
<!-- 올바름 -->
- 항목 1
  - 중첩된 항목 (올바른 들여쓰기)
- 항목 2
```

### 링크 참조

**문제**: 깨진 상대 링크
```markdown
<!-- 잘못됨 -->
[링크](../wrong/path.mdx)
```

**해결책**: 파일 경로 확인
```markdown
<!-- 올바름 -->
[링크](../correct/path.mdx)
```
