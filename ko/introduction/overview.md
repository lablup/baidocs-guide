# BaiDocs란 무엇인가

BaiDocs는 현대적인 웹 기술을 기반으로 구축된 포괄적인 문서화 플랫폼으로, 대규모 기술 문서의 생성 및 유지보수를 촉진하도록 설계되었습니다.

## 아키텍처 개요

BaiDocs는 다음으로 구성된 다중 애플리케이션 아키텍처를 사용합니다:

### 편집기 애플리케이션
다음 기능을 제공하는 웹 기반 콘텐츠 관리 인터페이스:
- 라이브 미리보기가 포함된 실시간 마크다운 편집
- Git 버전 관리와의 파일 시스템 통합
- 네비게이션 구조 관리
- 다국어 콘텐츠 조정
- 이미지 및 미디어 파일에 대한 자산 관리

### 뷰어 애플리케이션
다음 기능을 제공하는 최적화된 표현 계층:
- 최적의 성능을 위한 서버 사이드 렌더링
- 다양한 디바이스 유형을 위한 반응형 디자인
- 전문 텍스트 인덱싱을 통한 고급 검색 기능
- 테마 사용자 정의 및 브랜딩 옵션
- PDF 내보내기 기능

### 핵심 프레임워크
다음을 포함하는 기반 인프라:
- 향상된 마크다운 기능을 위한 MDX 처리 엔진
- 가상 디렉토리 구조를 위한 경로 매핑 시스템
- 버전 관리 워크플로를 위한 Git 통합
- 빌드 최적화 및 정적 사이트 생성
- 확장성을 위한 플러그인 아키텍처

## 기술 스택

BaiDocs는 다음을 사용하여 구축됩니다:

- **프론트엔드**: React 컴포넌트가 포함된 Next.js
- **스타일링**: Ant Design 컴포넌트가 포함된 Tailwind CSS
- **콘텐츠 처리**: remark/rehype 플러그인이 포함된 MDX
- **버전 관리**: simple-git 라이브러리와의 Git 통합
- **빌드 시스템**: 최적화된 캐싱이 포함된 Turbo 모노레포
- **배포**: 다양한 호스팅 옵션을 통한 정적 사이트 생성

## 사용 사례

BaiDocs는 다음과 같은 용도로 설계되었습니다:

### 기술 문서
- 소프트웨어 문서 및 API 참조
- 사용자 매뉴얼 및 운영 가이드
- 아키텍처 문서 및 설계 명세서

### 지식 관리
- 내부 위키 및 지식 베이스
- 프로세스 문서 및 표준 운영 절차
- 교육 자료 및 교육 콘텐츠

### 다중 팀 협업
- 기능 간 팀 문서
- 분산 팀 지식 공유
- 버전 관리 협업 편집

## 기타 솔루션과의 비교

기존 문서화 플랫폼과 달리 BaiDocs는 다음을 제공합니다:

- **Git 네이티브 워크플로**: 기존 개발 프로세스와의 직접 통합
- **모노레포 지원**: 복잡한 프로젝트 구조를 위한 가상 디렉토리 매핑
- **다국어 조정**: 여러 언어에 걸친 동기화된 콘텐츠 관리
- **개발자 친화적**: 프로그래밍 방식의 확장성을 갖춘 마크다운 우선 접근 방식

BaiDocs는 개발자 경험과 엔터프라이즈급 확장성에 중점을 두어 대안 솔루션과 차별화됩니다.
