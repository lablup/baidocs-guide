# 첫 번째 책 만들기

BaiDocs로 첫 번째 문서 책을 만들어보겠습니다! 이 가이드는 생성부터 게시까지 전체 과정을 안내합니다.

## 1단계: 에디터 접속

먼저 BaiDocs가 실행 중인지 확인합니다:

```bash
cd baidocs
pnpm dev
```

브라우저를 열고 **http://localhost:3001**로 이동합니다. BaiDocs 에디터 인터페이스를 볼 수 있습니다:

![BaiDocs 에디터 홈페이지](../_assets/editor-homepage.png)

에디터 홈페이지는 다음을 표시합니다:
- **기존 책들** (있는 경우)
- **새 책 생성** 버튼
- **저장소 복제** 옵션
- **가져오기 및 변환** 도구

## 2단계: 새 책 생성

**"New Book"** 버튼을 클릭하여 문서 생성을 시작합니다:

![새 책 모달](../_assets/new-book-modal.png)

### 기본 정보

책 세부 정보를 입력합니다:

- **Book Name**: `my-first-book` (디렉토리 이름으로 사용)
- **Display Title**: `My First Documentation`
- **Description**: 문서에 대한 간단한 설명
- **Emoji**: 책을 나타내는 이모지 선택 (선택사항)

### 언어 설정

- **Default Language**: `en` (영어)
- **Additional Languages**: 나중에 더 많은 언어를 추가할 수 있습니다

### 템플릿 선택

시작 템플릿을 선택합니다:

- **Empty**: 최소한의 구조로 시작
- **Documentation**: 기술 문서를 위한 미리 구축된 구조
- **API Reference**: API 문서를 위한 템플릿
- **Tutorial**: 단계별 가이드 템플릿

### Git 설정

버전 제어 처리 방법을 선택합니다:

#### 옵션 A: 로컬 Git 저장소

- **Git Type**: Local
- **Branch**: `main` (기본값)

나중에 원격에 연결할 수 있는 로컬 Git 저장소를 생성합니다.

#### 옵션 B: 원격 Git 저장소

- **Git Type**: Remote
- **Repository URL**: `https://github.com/your-username/my-docs.git`
- **Branch**: `main`

원격 저장소에 직접 연결합니다 (저장소가 존재해야 함).

## 3단계: 책 구조

생성되면 책은 다음 구조를 갖습니다:

```
content/my-first-book/
├── book.config.yaml     # 책 설정
├── en/                  # 영어 콘텐츠
│   ├── index.mdx       # 홈페이지
│   ├── getting-started/
│   │   └── README.md   # 시작 가이드
│   └── _assets/         # 에셋 디렉토리
└── .git/               # Git 저장소 (활성화된 경우)
```

### 책 설정

`book.config.yaml` 파일은 책의 동작을 제어합니다:

```yaml
id: my-first-book
title: My First Documentation
description: A brief description of your documentation
emoji: 📚
type: document
branch: main
languages:
  - en
defaultLanguage: en
navigation:
  en:
    - title: "Home"
      path: "index.mdx"
    - title: "Getting Started"
      path: "getting-started/README.md"
```

## 4단계: 콘텐츠 편집

### 책으로 이동

에디터에서 새로 생성된 책을 클릭하여 편집을 시작합니다:

![책 에디터 인터페이스](../_assets/book-editor-interface.png)

에디터 인터페이스에는 다음이 포함됩니다:

1. **파일 탐색기** (왼쪽 사이드바): 콘텐츠 파일 탐색
2. **에디터 패널** (중앙): 콘텐츠 작성 및 편집
3. **미리보기 패널** (오른쪽): 콘텐츠의 라이브 미리보기
4. **언어 선택기**: 언어 간 전환
5. **저장 버튼**: 변경사항 저장

### 홈페이지 편집

파일 탐색기에서 `index.mdx`를 클릭하여 홈페이지를 편집합니다:

```mdx
# 내 첫 번째 문서에 오신 것을 환영합니다

이것은 BaiDocs로 생성된 문서 책의 홈페이지입니다.

## 내용

- **시작하기**: 기본 사항을 학습합니다
- **고급 주제**: 기능에 대해 더 깊이 탐구합니다
- **API 참조**: 자세한 API 문서

## 기능

BaiDocs는 다음을 제공합니다:

- 📝 **쉬운 작성**: 마크다운/MDX로 작성
- 🌍 **다국어**: 여러 언어 지원
- 🔍 **검색**: 내장 검색 기능
- 📱 **반응형**: 모든 기기에서 작동

시작해보겠습니다!
```

### 새 콘텐츠 추가

다음과 같이 새 페이지를 생성합니다:

1. 파일 탐색기에서 우클릭
2. "New File" 선택
3. 파일명 입력: `advanced-features.mdx`
4. 콘텐츠 작성:

```mdx
# 고급 기능

이 페이지는 제품의 고급 기능을 다룹니다.

## 설정

다음을 사용하여 애플리케이션을 설정할 수 있습니다:

```yaml
# config.yaml
app:
  name: "My App"
  version: "1.0.0"
```

## API 통합

API에 연결합니다:

```javascript
const response = await fetch('/api/data');
const data = await response.json();
console.log(data);
```
```

### 내비게이션 업데이트

새 페이지를 포함하도록 `book.config.yaml`을 수정합니다:

```yaml
navigation:
  en:
    - title: "Home"
      path: "index.mdx"
    - title: "Getting Started"
      path: "getting-started/README.md"
    - title: "Advanced Features"
      path: "advanced-features.mdx"
```

## 5단계: 책 미리보기

### 에디터에서

에디터의 미리보기 패널을 사용하여 콘텐츠가 실시간으로 어떻게 보이는지 확인할 수 있습니다.

### 뷰어에서

**http://localhost:3000**을 열어 공개 버전을 확인합니다:

![뷰어 인터페이스](../_assets/viewer-interface.png)

뷰어는 다음을 표시합니다:
- 깔끔하고 반응형인 디자인
- 내비게이션 사이드바
- 검색 기능
- 언어 전환
- 모바일 친화적 레이아웃

## 6단계: 저장 및 커밋

### 변경사항 저장

에디터의 **저장** 버튼을 클릭하여 모든 변경사항을 저장합니다.

### Git에 커밋

Git을 활성화한 경우 변경사항을 커밋합니다:

```bash
cd content/my-first-book
git add .
git commit -m "Initial documentation setup"
```

### 원격에 푸시 (설정된 경우)

```bash
git push origin main
```
