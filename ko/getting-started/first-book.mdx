# 첫 번째 책 만들기

BaiDocs에서 첫 번째 문서 프로젝트를 생성해보세요. 이 단계별 가이드는 새 책 생성부터 첫 번째 콘텐츠 작성까지 모든 과정을 다룹니다.

## 새 책 생성

### 편집기에서 책 만들기

1. **편집기 시작**:
   ```bash
   npm run dev:editor
   ```

2. **브라우저에서 편집기 열기**:
   - http://localhost:3001로 이동

3. **새 책 만들기**:
   - 사이드바에서 "새 책 만들기" 버튼 클릭
   - 또는 `+` 아이콘 클릭

![새 책 만들기 인터페이스](../_assets/create-new-book-ko.png)

### 책 정보 설정

새 책 생성 대화상자에서 다음 정보를 입력하세요:

#### 기본 정보

**책 ID** (필수):
```
my-first-book
```
- 소문자, 하이픈, 숫자만 허용
- URL과 파일 경로에서 사용됩니다
- 예: `my-first-book`, `api-guide-v2`, `user-manual`

**제목** (필수):
```
나의 첫 번째 문서
```
- 사용자에게 표시되는 친화적인 제목
- 한국어와 특수문자 사용 가능

**설명** (선택사항):
```
BaiDocs를 사용한 첫 번째 문서 프로젝트
```

**이모지** (선택사항):
```
📚
```
- 책을 시각적으로 구분하는 데 도움
- 네비게이션과 목록에서 표시됩니다

#### 언어 설정

**지원 언어**:
- ✅ 한국어 (ko)
- ✅ English (en)

**기본 언어**:
```
한국어 (ko)
```

#### 고급 옵션

**Git 저장소** (선택사항):
- 나중에 설정 또는 기존 저장소 연결 가능
- 협업과 버전 관리에 필요

**템플릿**:
- 📖 문서 (기본값)
- 📘 API 참조
- 📗 튜토리얼
- 📙 사용자 가이드

### 생성 확인

"책 생성" 버튼을 클릭하면:

1. **디렉토리 생성**: `content/my-first-book/` 폴더 생성
2. **구성 파일 생성**: `book.config.yaml` 파일 생성
3. **초기 페이지 생성**: 각 언어의 `index.mdx` 파일 생성
4. **Git 초기화**: Git 저장소 초기화 (선택사항)

## 생성된 구조 탐색

### 파일 구조

생성된 책의 구조를 확인해보세요:

```
content/my-first-book/
├── book.config.yaml          # 책 설정 파일
├── ko/                      # 한국어 콘텐츠
│   ├── index.mdx            # 한국어 홈페이지
│   └── _assets/              # 한국어 이미지
├── en/                      # 영어 콘텐츠
│   ├── index.mdx            # 영어 홈페이지
│   └── _assets/              # 영어 이미지
└── .git/                    # Git 저장소 (선택사항)
```

### 구성 파일 이해

`book.config.yaml` 파일에는 책의 모든 설정이 포함됩니다:

```yaml
id: my-first-book
title: 나의 첫 번째 문서
description: BaiDocs를 사용한 첫 번째 문서 프로젝트
emoji: 📚
type: document
branch: main

# 지원 언어
languages:
  - ko
  - en

# 기본 언어
defaultLanguage: ko

# 네비게이션 구조
navigation:
  ko:
    - title: "홈"
      path: "index.mdx"
  en:
    - title: "Home"
      path: "index.mdx"

# Git 설정 (선택사항)
git:
  repository: ""
  branch: main
```

## 첫 번째 콘텐츠 작성

### 홈페이지 편집

1. **편집기에서 파일 열기**:
   - 사이드바에서 "my-first-book" 확장
   - "ko" 폴더 확장
   - "index.mdx" 파일 클릭

2. **초기 콘텐츠 확인**:
   ```mdx
   # 나의 첫 번째 문서

   BaiDocs에 오신 것을 환영합니다!

   이곳에서 문서 작성을 시작하세요.
   ```

3. **콘텐츠 개선**:
   ```mdx
   # 나의 첫 번째 문서

   BaiDocs를 사용한 첫 번째 문서 프로젝트에 오신 것을 환영합니다!

   ## 소개

   이 문서는 BaiDocs의 강력한 기능을 보여주는 예제입니다.
   Markdown과 React 컴포넌트를 자유롭게 조합하여 풍부한 콘텐츠를 만들 수 있습니다.

   ## 주요 기능

   ### 📝 Markdown 지원

   일반적인 Markdown 문법을 모두 지원합니다:

   - **굵은 텍스트**와 *기울임 텍스트*
   - [링크](https://baidocs.com)와 `인라인 코드`
   - 순서가 있는 목록과 없는 목록

   ### ⚡ React 컴포넌트

   MDX의 강력함으로 React 컴포넌트를 직접 사용할 수 있습니다:

   <div style={{
     backgroundColor: '#e3f2fd',
     padding: '1rem',
     borderRadius: '8px',
     border: '1px solid #90caf9'
   }}>
     🎉 이것은 사용자 정의 스타일이 적용된 React 컴포넌트입니다!
   </div>

   ### 📊 코드 블록

   구문 강조가 적용된 코드 블록:

   ```javascript
   function welcome(name) {
     console.log(`안녕하세요, ${name}님!`);
     return `BaiDocs에 오신 것을 환영합니다!`;
   }

   welcome("개발자");
   ```

   ```python
   def welcome(name):
       print(f"안녕하세요, {name}님!")
       return "BaiDocs에 오신 것을 환영합니다!"

   welcome("개발자")
   ```

   ## 다음 단계

   이제 문서 작성을 시작할 준비가 되었습니다:

   1. 새 페이지 추가하기
   2. 네비게이션 구조 설정하기
   3. 이미지와 에셋 추가하기
   4. Git 저장소 연결하기

   ---

   *더 많은 기능을 알아보려면 [BaiDocs 가이드](/baidocs-guide)를 참고하세요.*
   ```

### 실시간 미리보기

편집기에서 콘텐츠를 수정하면 우측 패널에서 실시간으로 미리보기를 확인할 수 있습니다:

- 자동 저장으로 변경사항이 즉시 반영됩니다
- 구문 오류나 렌더링 문제를 즉시 확인할 수 있습니다

```yaml

      path: "index.mdx"
    - title: "시작하기"
      path: "getting-started.mdx"
  en:
    - title: "Home"
      path: "index.mdx"
    - title: "Getting Started"
      path: "getting-started.mdx"
```

## 이미지 추가

### 이미지 업로드

1. **이미지 디렉토리로 이동**:
   - `ko/_assets/` 폴더 클릭

2. **이미지 업로드**:
   - 드래그 앤 드롭 또는 업로드 버튼 클릭
   - 지원 형식: PNG, JPG, GIF, SVG, WebP

3. **이미지 사용**:
   ```mdx
   # 이미지가 포함된 페이지

   다음은 우리 제품의 스크린샷입니다:

   ![제품 스크린샷](_assets/product-screenshot.png)

   ## 다이어그램

   아키텍처 다이어그램:

   ![아키텍처](_assets/architecture-diagram.svg)
   ```

### 이미지 최적화 팁

- **적절한 크기**: 웹에 적합한 해상도 (보통 1200px 이하)
- **파일 형식**:
  - 사진: JPG (손실 압축)
  - 그래픽/로고: PNG (무손실)
  - 아이콘/단순 그래픽: SVG (벡터)
- **파일 이름**: 영문, 숫자, 하이픈만 사용
- **Alt 텍스트**: 접근성을 위한 설명 텍스트 포함

## Git 저장소 연결

### 로컬 Git 저장소 초기화

```bash
# 책 디렉토리로 이동
cd content/my-first-book

# Git 저장소 초기화 (아직 되지 않은 경우)
git init

# 첫 번째 커밋
git add .
git commit -m "첫 번째 BaiDocs 문서 생성

- 기본 페이지 구조 설정
- 홈페이지와 시작하기 페이지 추가
- 한국어/영어 다국어 지원 설정"
```

### 원격 저장소 연결

```bash
# GitHub에서 새 저장소 생성 후

# 원격 저장소 추가
git remote add origin https://github.com/username/my-first-book.git

# 첫 번째 푸시
git branch -M main
git push -u origin main
```

### book.config.yaml 업데이트

원격 저장소 정보를 설정에 추가:

```yaml
# Git 설정 업데이트
git:
  repository: "https://github.com/username/my-first-book.git"
  branch: main
```

## 팁과 모범 사례

### 콘텐츠 구성

- **논리적 구조**: 정보를 논리적 순서로 구성

### 파일 관리

- **의미 있는 이름**: 파일과 폴더에 설명적인 이름 사용
- **일관된 명명**: kebab-case 사용 (예: `getting-started.mdx`)
- **적절한 구조**: 콘텐츠 유형별로 폴더 구성
