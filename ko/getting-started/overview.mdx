# 개요

BaiDocs는 콘텐츠 생성과 콘텐츠 표현을 분리하여 모든 규모의 팀에게 유연하고 강력한 솔루션을 제공하는 현대적인 문서화 프레임워크입니다.

## 아키텍처 개요

BaiDocs는 편집과 보기 사이의 관심사를 분리하는 **이중 애플리케이션 아키텍처**를 따릅니다:

### 📝 **에디터 애플리케이션** (`/apps/editor`)

에디터는 다음 기능을 제공하는 Next.js 애플리케이션입니다:

- **콘텐츠 관리**: 문서 생성, 편집, 구성
- **Git 통합**: 원격 저장소 연결, 브랜치 관리
- **가져오기 도구**: Sphinx, MkDocs, GitBook 형식에서 변환
- **라이브 미리보기**: 콘텐츠 변경사항의 실시간 미리보기
- **다국어 지원**: 번역 및 언어별 콘텐츠 관리

**주요 사용 사례**: 콘텐츠 작성자와 문서 관리자가 로컬에서 에디터를 사용하여 문서를 생성하고 관리합니다.

### 👀 **뷰어 애플리케이션** (`/apps/viewer`)

뷰어는 공개 사용에 최적화된 Next.js 애플리케이션입니다:

- **빠른 렌더링**: 성능 및 SEO에 최적화
- **반응형 디자인**: 모바일 우선의 접근 가능한 인터페이스
- **검색 통합**: 모든 콘텐츠에 대한 전문 검색
- **버전 지원**: 다양한 버전/브랜치 탐색
- **배포 준비**: Vercel, Netlify 등에 쉬운 배포

**주요 사용 사례**: 최종 사용자와 문서 소비자가 문서 읽기를 위해 뷰어에 액세스합니다.

## 프로젝트 구조

```
baidocs/
├── apps/
│   ├── editor/          # 콘텐츠 생성 및 관리
│   └── viewer/          # 공개 문서 사이트
├── content/             # 문서 책들
│   ├── book1/
│   │   ├── book.config.yaml
│   │   ├── en/          # 영어 콘텐츠
│   │   └── ko/          # 한국어 콘텐츠
│   └── book2/
├── packages/
│   └── ui/              # 공유 UI 구성 요소
└── ...
```

### 콘텐츠 구성

각 문서 책은 다음 구조를 따릅니다:

```
content/your-book/
├── book.config.yaml     # 책 설정
├── en/                  # 영어 콘텐츠
│   ├── index.mdx       # 홈페이지
│   ├── getting-started/
│   └── _assets/         # 에셋
├── ko/                 # 한국어 콘텐츠 (선택사항)
│   ├── index.mdx
│   └── ...
└── .git/               # Git 저장소 (선택사항)
```

## Git 브랜치를 통한 버전 제어

BaiDocs는 강력한 버전 관리를 위해 Git 브랜치를 활용합니다:

### 🌿 **브랜치 기반 버전 관리**

- **다중 버전**: 각 Git 브랜치가 서로 다른 버전을 나타냄
- **라이브 전환**: 독자가 뷰어에서 버전 간 전환 가능
- **독립적 콘텐츠**: 각 브랜치가 서로 다른 콘텐츠 구조를 가질 수 있음

```
main          ← 최신 안정 버전
├── v2.0      ← 버전 2.0 문서
├── v1.0      ← 버전 1.0 문서
└── dev       ← 개발/미리보기 버전
```

### 🌍 **언어 지원**

BaiDocs는 네이티브 다국어 지원을 제공합니다:

- **언어별 디렉토리**: 각 언어가 고유한 폴더를 가짐 (`en/`, `ko/`, `fr/`)
- **공유 설정**: `book.config.yaml`의 내비게이션 및 메타데이터
- **독립적 번역**: 각 언어가 서로 다른 콘텐츠 구조를 가질 수 있음
- **언어 전환**: 뷰어에서 자동 언어 감지 및 전환

## MDX: 향상된 마크다운

BaiDocs는 **MDX** (Markdown + JSX)를 주요 콘텐츠 형식으로 사용합니다:

### 📄 **표준 마크다운**
모든 표준 마크다운 문법이 지원됩니다:
- 헤더, 목록, 링크, 이미지
- 구문 강조가 있는 코드 블록
- 표, 인용문

### ⚡ **MDX 확장**
- **React 구성 요소**: 인터랙티브 구성 요소 포함
- **사용자 정의 스타일링**: 고급 레이아웃을 위한 JSX 사용
- **동적 콘텐츠**: 콘텐츠 내 JavaScript 표현식

```mdx
# 일반 마크다운 헤더

이것은 일반 마크다운 콘텐츠입니다.

<div style={{backgroundColor: '#f0f0f0', padding: '1rem'}}>
  이것은 **마크다운**과 혼합된 JSX 콘텐츠입니다.
</div>

import CustomComponent from './components/CustomComponent'

<CustomComponent data="example" />
```

### 🎨 **BaiDocs 확장**
- **어드모니션**: 메모, 경고, 팁을 위한 특별한 콜아웃 박스
- **코드 탭**: 다국어 코드 예제
- **이미지 최적화**: 자동 이미지 처리 및 최적화

## 가져오기 및 변환 기능

BaiDocs는 인기 있는 형식에서 문서를 가져오고 변환할 수 있습니다:

### 📚 **지원되는 형식**

| 형식 | 가져오기 지원 | 변환 기능 |
|--------|---------------|-------------------|
| **Sphinx** | ✅ 전체 | RST → MDX, 내비게이션, 에셋 |
| **MkDocs** | ✅ 전체 | YAML 설정, 테마, 확장 |
| **GitBook** | ✅ 전체 | GitBook 형식 → BaiDocs 구조 |
| **일반 마크다운** | ✅ 기본 | 파일 구조 보존 |

### 🧹 **깔끔한 가져오기 프로세스**

문서를 가져올 때:

1. **분석**: BaiDocs가 소스 구조를 분석
2. **변환**: 콘텐츠가 MDX 형식으로 변환
3. **깔끔한 저장소**: 새롭고 깔끔한 Git 저장소 생성
4. **구조 최적화**: BaiDocs 형식으로 콘텐츠 구성
5. **에셋 마이그레이션**: 이미지와 파일이 적절히 구성

결과는 원래 도구의 부담 없이 깔끔하고 최적화된 BaiDocs 저장소입니다.

## 모노레포 지원

BaiDocs는 모노레포 설정을 자동으로 감지하고 지원합니다:

### 🏢 **자동 감지**

BaiDocs는 문서가 더 큰 저장소의 일부인지 자동으로 감지합니다:

```
your-monorepo/
├── src/                 # 애플리케이션 코드
├── docs/               # BaiDocs 콘텐츠 (자동 감지!)
│   ├── book.config.yaml
│   ├── en/
│   └── ko/
└── package.json
```

### 🔧 **설정**

모노레포의 경우, BaiDocs는 다음과 같이 설정할 수 있습니다:
- **하위 디렉토리 경로**: 문서가 위치하는 곳 지정
- **빌드 통합**: 기존 CI/CD 파이프라인과 통합
- **선택적 배포**: 문서 변경사항만 배포
