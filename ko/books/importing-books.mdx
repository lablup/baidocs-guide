# 가져오기 & 문서 변환

BaiDocs는 인기 있는 문서화 도구에서 문서를 가져와 변환할 수 있으며, 기존 콘텐츠에서 깔끔하고 최적화된 BaiDocs 저장소를 생성합니다.

## 지원되는 가져오기 소스

| 도구 | 가져오기 지원 | 변환 기능 |
|------|-------------|----------|
| **Sphinx** | ✅ 완전 지원 | RST → MDX, 네비게이션, 설정, 자산 |
| **MkDocs** | ✅ 완전 지원 | Markdown → MDX, mkdocs.yml → book.config.yaml |
| **GitBook** | ✅ 완전 지원 | GitBook 형식 → BaiDocs 구조 |
| **일반 Markdown** | ✅ 기본 지원 | 파일 구조 보존, 기본 네비게이션 |
| **Confluence** | 🔄 곧 지원 | 내보내기 → MDX 변환 |
| **Notion** | 🔄 곧 지원 | 데이터베이스 → 문서 구조 |

## 가져오기 과정 개요

가져오기 & 변환 과정은 완전히 새롭고 깔끔한 BaiDocs 저장소를 생성합니다:

```
원본 도구 저장소                    BaiDocs 저장소
├── sphinx/                      ├── book.config.yaml
│   ├── conf.py                 ├── en/
│   ├── index.rst               │   ├── index.mdx
│   ├── _build/                 │   ├── getting-started/
│   └── ...                     │   └── _assets/
                                └── .git/ (깔끔한 히스토리)
```

### 주요 장점

✅ **깔끔한 저장소**: 레거시 아티팩트 없이 새로운 Git 히스토리
✅ **최적화된 구조**: BaiDocs 네이티브 구성
✅ **자산 마이그레이션**: 이미지와 파일이 적절히 구성됨
✅ **형식 변환**: 향상된 기능을 가진 최신 MDX
✅ **네비게이션 생성**: 지능적인 네비게이션 구조

## Sphinx에서 가져오기

### 준비

Sphinx 프로젝트에 다음이 있는지 확인하세요:
- `conf.py` 설정 파일
- `.rst` 파일로 구성된 구조화된 문서
- 빌드된 HTML 버전 (참조용)

### 가져오기 설정

![Sphinx 가져오기 설정](../_assets/sphinx-import-config.png)

#### 소스 설정

- **저장소 URL**: `https://github.com/your-org/sphinx-docs.git`
- **브랜치**: 소스 브랜치 (보통 `main` 또는 `master`)
- **형식**: "Sphinx" 선택

#### 대상 설정

- **대상 이름**: 새로운 BaiDocs 책 이름 (예: `api-documentation`)
- **표시 제목**: 사람이 읽을 수 있는 제목
- **이모지**: 책의 시각적 식별자

#### 변환 옵션

- **언어 매핑**: Sphinx 언어를 BaiDocs에 매핑
  ```
  Sphinx: en → BaiDocs: en
  Sphinx: es → BaiDocs: es
  ```

- **콘텐츠 처리**:
  - ✅ RST를 MDX로 변환
  - ✅ toctree에서 네비게이션 추출
  - ✅ 이미지와 자산 마이그레이션
  - ✅ 구문 하이라이팅이 있는 코드 블록 변환

### 변환 결과

**이전 (Sphinx)**:
```python
# conf.py
project = 'My API Documentation'
html_theme = 'sphinx_rtd_theme'
extensions = ['sphinx.ext.autodoc']
```

```rst
.. index.rst
My API Documentation
====================

Welcome to the API documentation.

.. toctree::
   :maxdepth: 2

   getting-started
   api-reference
```

**이후 (BaiDocs)**:
```yaml
# book.config.yaml
id: api-documentation
title: My API Documentation
description: Sphinx에서 변환됨
navigation:
  en:
    - title: "홈"
      path: "index.mdx"
    - title: "시작하기"
      path: "getting-started.mdx"
    - title: "API 참조"
      path: "api-reference.mdx"
```

```mdx
# index.mdx
# My API Documentation

Welcome to the API documentation.

이 문서는 Sphinx에서 BaiDocs 형식으로 변환되었습니다.
```

## MkDocs에서 가져오기

### 준비

MkDocs 프로젝트에 다음이 포함되어 있는지 확인하세요:
- `mkdocs.yml` 설정 파일
- `docs/` 디렉토리의 Markdown 파일
- 적절히 구조화된 네비게이션

### 가져오기 설정

#### 소스 분석

BaiDocs는 MkDocs 설정을 분석합니다:

```yaml
# mkdocs.yml (원본)
site_name: 사용자 가이드
site_description: 완전한 사용자 문서
nav:
  - 홈: index.md
  - 시작하기:
    - 설치: getting-started/install.md
    - 빠른 시작: getting-started/quickstart.md
  - 고급: advanced.md
```

#### 변환 과정

- **설정 마이그레이션**: `mkdocs.yml` → `book.config.yaml`
- **콘텐츠 변환**: Markdown → MDX (향상된 기능)
- **테마 번역**: MkDocs 테마 → BaiDocs 스타일링
- **플러그인 변환**: 일반적인 플러그인 → BaiDocs 동등물

### 변환 결과

**변환 후**:
```yaml
# book.config.yaml
id: user-guide
title: 사용자 가이드
description: 완전한 사용자 문서
navigation:
  en:
    - title: "홈"
      path: "index.mdx"
    - title: "시작하기"
      items:
        - title: "설치"
          path: "getting-started/install.mdx"
        - title: "빠른 시작"
          path: "getting-started/quickstart.mdx"
    - title: "고급"
      path: "advanced.mdx"
```

### 플러그인 변환

일반적인 MkDocs 플러그인과 BaiDocs 동등물:

| MkDocs 플러그인 | BaiDocs 기능 |
|----------------|-------------|
| `search` | 내장 검색 |
| `admonitions` | 네이티브 어드모니션 |
| `codehilite` | 구문 하이라이팅 |
| `toc` | 자동 목차 |
| `mkdocs-material` | 내장 반응형 테마 |

## GitBook에서 가져오기

### GitBook 레거시 형식

GitBook Legacy (v1) 저장소의 경우:

```
gitbook-repo/
├── SUMMARY.md          # GitBook 네비게이션
├── README.md          # 홈페이지
├── chapter1/
│   ├── README.md
│   └── section1.md
└── assets/
    └── _assets/
```

### GitBook V2 형식

최신 GitBook 내보내기의 경우:

```
gitbook-export/
├── .gitbook.yaml      # GitBook 설정
├── README.md
├── getting-started/
└── reference/
```

### 변환 과정

#### 네비게이션 변환

**GitBook SUMMARY.md**:
```markdown
# 요약

* [소개](README.md)
* [시작하기](getting-started/README.md)
  * [설치](getting-started/installation.md)
  * [설정](getting-started/configuration.md)
* [고급 주제](advanced/README.md)
```

**BaiDocs book.config.yaml**:
```yaml
navigation:
  en:
    - title: "소개"
      path: "index.mdx"
    - title: "시작하기"
      items:
        - title: "설치"
          path: "getting-started/installation.mdx"
        - title: "설정"
          path: "getting-started/configuration.mdx"
    - title: "고급 주제"
      path: "advanced/index.mdx"
```

#### 콘텐츠 향상

변환 중에 BaiDocs는 GitBook 콘텐츠를 향상시킵니다:

- **힌트 블록** → **어드모니션**
- **코드 탭** → **MDX 코드 컴포넌트**
- **임베디드 콘텐츠** → **React 컴포넌트**
- **GitBook 변수** → **MDX 변수**

## 일반 Markdown 가져오기

특정 문서화 도구 없이 Markdown 파일이 있는 저장소의 경우:

### 자동 구조 감지

BaiDocs는 저장소 구조를 분석합니다:

```
generic-docs/
├── README.md          # index.mdx가 됨
├── docs/
│   ├── guide.md
│   ├── tutorial.md
│   └── reference/
│       └── api.md
└── _assets/
```

### 네비게이션 생성

BaiDocs는 다음을 기반으로 자동으로 네비게이션을 생성합니다:

1. **파일 구조**: 디렉토리 계층
2. **프론트매터**: 파일의 YAML 메타데이터
3. **헤더**: 파일의 첫 번째 수준 헤더
4. **파일 이름**: 의미있는 파일 이름

**생성된 네비게이션**:
```yaml
navigation:
  en:
    - title: "홈"
      path: "index.mdx"
    - title: "가이드"
      path: "guide.mdx"
    - title: "튜토리얼"
      path: "tutorial.mdx"
    - title: "참조"
      items:
        - title: "API"
          path: "reference/api.mdx"
```

## 고급 가져오기 옵션

### 대상 구조 선택

가져온 콘텐츠를 구성하는 방법을 선택하세요:

#### 독립형 구조
```
content/imported-book/
├── book.config.yaml
├── en/
└── .git/
```

#### 모노레포 구조
```
content/imported-book/
├── docs/              # 하위 디렉토리
│   ├── book.config.yaml
│   ├── en/
│   └── ko/
└── .git/
```

### 콘텐츠 처리 옵션

#### 형식 변환

- **Markdown → MDX**: JSX 기능으로 향상
- **RST → MDX**: Sphinx reStructuredText 변환
- **HTML → MDX**: HTML 소스에서 깔끔한 변환

#### 자산 처리

- **이미지 최적화**: 자동 이미지 압축 및 최적화
- **경로 수정**: 상대 경로 및 참조 수정
- **자산 구성**: 이미지와 파일 중앙화

#### 콘텐츠 향상

- **구문 하이라이팅**: 코드 블록 업그레이드
- **어드모니션**: 콜아웃과 노트 박스 변환
- **상호 참조**: 내부 링크 유지
- **테이블 향상**: 테이블 형식 개선

## 가져오기 후 최적화

가져오기 후 BaiDocs는 정리 및 최적화를 수행합니다:

### 저장소 정리

1. **Git 히스토리 정리**: 깔끔한 커밋 히스토리를 가진 새 저장소
2. **빌드 아티팩트 제거**: `_build/`, `site/` 또는 기타 생성된 파일 없음
3. **구조 최적화**: BaiDocs 네이티브 구성
4. **의존성 업데이트**: 최신 도구체인과 의존성

### 콘텐츠 최적화

1. **네비게이션 정제**: 논리적인 네비게이션 구조
2. **콘텐츠 검증**: 깨진 링크와 참조 확인
3. **형식 표준화**: 전체적으로 일관된 형식
4. **자산 최적화**: 압축된 이미지와 최적화된 미디어

### 설정 향상

```yaml
# 가져오기 후 향상된 book.config.yaml
id: imported-documentation
title: 가져온 문서
description: [원본 도구]에서 BaiDocs로 변환됨
emoji: 🔄
type: document
languages:
  - en
defaultLanguage: en
# 변환 메타데이터
conversion:
  source: sphinx
  originalRepo: https://github.com/org/original-docs
  convertedAt: 2024-01-26T00:00:00Z
navigation:
  # 생성된 네비게이션 구조
```

## 가져오기 문제 해결

### 일반적인 가져오기 문제

#### "지원되지 않는 소스 형식"
- 저장소에 지원되는 형식의 문서가 포함되어 있는지 확인
- 필요한 설정 파일(conf.py, mkdocs.yml 등) 확인
- 일반 Markdown 가져오기 사용 고려

#### "네비게이션 추출 실패"
- 소스 네비게이션이 복잡하거나 비표준일 수 있음
- 가져오기 후 수동 네비게이션 편집이 필요할 수 있음
- 네비게이션 파일의 오류 확인

#### "자산 변환 오류"
- 일부 자산이 지원되지 않는 형식일 수 있음
- 소스 문서의 자산 경로 검토
- 수동 자산 마이그레이션이 필요할 수 있음

### 복구 및 수동 수정

#### 가져오기 후 수동 편집

1. **생성된 네비게이션 검토**: 필요에 따라 구조 조정
2. **깨진 링크 수정**: 올바르게 변환되지 않은 참조 업데이트
3. **콘텐츠 향상**: BaiDocs 특화 기능 추가
4. **모든 페이지 테스트**: 콘텐츠가 올바르게 렌더링되는지 확인

#### 다른 설정으로 재가져오기

```bash
# 실패한 가져오기 제거
rm -rf content/failed-import

# 다른 설정으로 시도:
# - 다른 대상 구조
# - 수정된 소스 브랜치
# - 대체 형식 감지
```

## 모범 사례

### 가져오기 전

- **원본 백업**: 원본 문서가 안전하게 백업되었는지 확인
- **소스 정리**: 불필요한 파일과 아티팩트 제거
- **과정 문서화**: 사용자 정의 설정이나 특별한 요구사항 기록

### 가져오기 중

- **진행 상황 모니터링**: 경고나 오류에 대한 가져오기 로그 확인
- **설정 검토**: 대상 설정이 올바른지 확인
- **점진적 테스트**: 복잡한 프로젝트의 경우 작은 섹션부터 가져오기

### 가져오기 후

- **콘텐츠 검토**: 변환된 콘텐츠를 수동으로 검토
- **링크 확인**: 모든 내부 및 외부 링크 확인
- **네비게이션 테스트**: 네비게이션이 올바르게 작동하는지 확인
- **팀 검토**: 팀 멤버가 변환된 콘텐츠를 검토하도록 함
