# 기본 마크다운 문법

BaiDocs는 GitHub Flavored Markdown (GFM) 확장 기능과 함께 표준 마크다운 문법을 지원합니다.

## 제목

`#` 기호를 사용하여 제목 생성:

```markdown
# 제목 1
## 제목 2
### 제목 3
```

:::tip
페이지당 하나의 H1 (`#`)만 사용하세요 - 페이지 제목이어야 합니다.
:::

## 단락

빈 줄로 단락 분리:

```markdown
첫 번째 단락입니다.

두 번째 단락입니다.
```

## 강조

### 굵게

```markdown
**굵은 텍스트**
```

**굵은 텍스트**

### 기울임

```markdown
*기울임 텍스트*
```

*기울임 텍스트*

### 취소선

```markdown
~~취소선 텍스트~~
```

~~취소선 텍스트~~

## 리스트

### 순서 없는 리스트

```markdown
- 첫 번째 항목
- 두 번째 항목
  - 중첩된 항목
- 세 번째 항목
```

- 첫 번째 항목
- 두 번째 항목
  - 중첩된 항목
- 세 번째 항목

### 순서 있는 리스트

```markdown
1. 첫 번째 항목
2. 두 번째 항목
3. 세 번째 항목
```

1. 첫 번째 항목
2. 두 번째 항목
3. 세 번째 항목

### 작업 리스트

```markdown
- [x] 완료된 작업
- [ ] 미완료 작업
```

- [x] 완료된 작업
- [ ] 미완료 작업

## 링크

### 인라인 링크

```markdown
[링크 텍스트](https://example.com)
```

[링크 텍스트](https://example.com)

### 내부 링크

```markdown
[설치 가이드](installation.md)
[이 페이지의 섹션](#headings)
```

## 이미지

```markdown
![대체 텍스트](images/filename.png)
```

## 인라인 코드

```markdown
`useState` 훅은 React에서 상태 관리에 사용됩니다.
```

`useState` 훅은 React에서 상태 관리에 사용됩니다.

## 프론트매터

모든 페이지는 YAML 프론트매터로 시작해야 합니다:

```markdown
---
title: 페이지 제목
description: 검색 및 SEO를 위한 페이지 설명
---

# 페이지 콘텐츠는 여기서 시작
```
