# 마크다운 기초

마크다운은 BaiDocs에서 모든 콘텐츠 생성의 기반이 됩니다. 이 섹션에서는 기술 문서화를 위한 필수 마크다운 문법과 모범 사례를 다룹니다.

## 문서 구조

### 제목과 계층구조

제목을 사용하여 문서 구조와 네비게이션 계층구조를 설정하십시오:

```markdown
# H1 - 문서 제목
## H2 - 주요 섹션
### H3 - 하위 섹션
#### H4 - 세부 주제
##### H5 - 특정 포인트
###### H6 - 세부 사항
```

#### 제목 가이드라인

- **단일 H1**: 주제목에는 문서당 하나의 H1만 사용
- **논리적 순서**: 레벨을 건너뛰지 않고 순차적인 제목 레벨 사용
- **서술적 제목**: 명확하고 서술적인 제목 텍스트 사용
- **일관된 형식**: 문서 전체에서 균일한 제목 스타일 유지

### 문단 구조

```markdown
빈 줄로 텍스트를 구분하여 일반적인 문단을 생성합니다.
이 텍스트는 같은 문단을 계속합니다.

빈 줄 후에 이 텍스트는 새로운 문단을 시작합니다.

가독성 향상을 위해 문단을 단일 개념이나 아이디어에 집중하여 작성하십시오.
긴 문단은 작고 소화하기 쉬운 섹션으로 나누십시오.
```

## 텍스트 형식

### 기본 텍스트 스타일

```markdown
**굵은 텍스트**는 강조와 중요성을 위해 사용
*기울임 텍스트*는 미묘한 강조나 기술 용어에 사용
***굵은 기울임***은 최대한의 강조를 위해 사용
~~취소선~~은 폐지되거나 잘못된 정보에 사용
```

### 인라인 코드와 기술 용어

```markdown
다음 용도로 `인라인 코드`를 사용하십시오:
- 함수명: `getUserData()`
- 파일명: `config.yaml`
- 명령줄 도구: `npm install`
- 변수명: `API_KEY`
- 짧은 코드 조각: `const result = true`
```

## 목록과 구성

### 순서 없는 목록

```markdown
- 주요 항목
- 보조 항목
  - 중첩 항목
  - 다른 중첩 항목
    - 깊게 중첩된 항목
- 주요 레벨로 돌아감

대체 문법:
* 주요 항목
* 보조 항목
+ 혼합 글머리 스타일 (권장하지 않음)
```

### 순서 있는 목록

```markdown
1. 프로세스의 첫 번째 단계
2. 순서의 두 번째 단계
   1. 하위 단계 1
   2. 하위 단계 2
3. 세 번째 단계 계속
4. 마지막 단계

대체 번호 매기기:
1. 첫 번째 항목
1. 두 번째 항목 (자동 번호)
1. 세 번째 항목 (자동 번호)
```

### 작업 목록

```markdown
- [x] 완료된 작업
- [ ] 미완성 작업
- [x] 또 다른 완료된 작업
- [ ] 향후 작업
```

## 링크와 참조

### 기본 링크 문법

```markdown
[링크 텍스트](https://example.com)
[제목이 있는 링크](https://example.com "호버 제목")
[상대 링크](../other-section/page.md)
[내부 앵커](#section-header)
```

### 참조 스타일 링크

```markdown
이 문단은 [참조 링크][ref1]와 [다른 참조][ref2]를 포함합니다.

[ref1]: https://example.com "참조 제목"
[ref2]: ./local-file.md "로컬 파일 참조"
```

### 자동 링크

```markdown
직접 URL: https://example.com
이메일 주소: contact@company.com
```

## 특수 문자 이스케이프

### 문자 이스케이프

```markdown
\*기울임 텍스트가 아님\*
\[링크가 아님\]
\`인라인 코드가 아님\`
\# 제목이 아님

백슬래시(\)를 사용하여 이스케이프:
* 별표
_ 밑줄
# 해시 기호
` 백틱
[ ] 대괄호
( ) 소괄호
```

### HTML 문자 엔터티

```markdown
&lt; 보다 작음
&gt; 보다 큼
&amp; 앰퍼샌드
&quot; 따옴표
&copy; 저작권 기호
```

## 문서 메타데이터

```markdown
date: 2026-01-20
tags:
  - documentation
  - guide
  - markdown
```

```markdown
<!-- 검토자: 홍길동 -->
<!-- 버전: 1.2 -->
```

## 모범 사례

### 일관성 가이드라인

#### 형식 표준

- **제목**: 제목에는 문장 대소문자 사용 (제목 대소문자 아님)
- **목록**: 순서 없는 목록에 일관되게 하이픈(-) 사용
- **강조**: 중요한 정보에는 굵게, 정의에는 기울임 사용
- **링크**: 서술적인 링크 텍스트 사용, "여기를 클릭" 또는 "더 읽기" 피하기

#### 콘텐츠 구성

- **섹션당 단일 주제**: 섹션을 특정 개념에 집중하여 작성
- **논리적 흐름**: 일반적인 것에서 구체적인 것으로 콘텐츠 구성
- **명확한 전환**: 섹션 간에 연결 문장 사용
- **스캔 가능한 구조**: 쉬운 스캔을 위해 제목과 목록 사용

### 접근성 고려사항

#### 스크린 리더 호환성

- **의미론적 제목**: 스크린 리더를 위한 적절한 제목 계층구조 사용
- **대체 텍스트**: 이미지에 의미 있는 대체 텍스트 제공 (미디어 섹션에서 다룸)
- **링크 맥락**: 주변 맥락 없이도 링크 텍스트가 의미 있도록 보장
- **목록 구조**: 네비게이션 보조를 위한 적절한 목록 마크업 사용

#### 국제적 고려사항

- **언어 지정**: 문서 메타데이터에 언어 코드 포함
- **문화적 민감성**: 예시와 참조에서 문화적 차이 고려
- **번역 준비**: 번역을 위해 명확하고 명료한 텍스트 작성
- **유니코드 지원**: 국제 콘텐츠를 위한 적절한 유니코드 문자 사용

### 성능 최적화

#### 문서 길이

- **적절한 크기**: 개별 문서를 10,000단어 미만으로 유지
- **논리적 구분**: 긴 콘텐츠를 여러 관련 문서로 분할
- **상호 참조**: 링크를 사용하여 문서 간 관련 콘텐츠 연결

#### 파일 구성

- **서술적 이름**: 명확하고 서술적인 파일명 사용
- **일관된 구조**: 균일한 디렉토리 구성 유지
- **버전 관리**: 콘텐츠 변경을 위한 버전 관리 모범 사례 준수

### 품질 보증

#### 콘텐츠 검토 프로세스

1. **문법 및 맞춤법**: 자동화된 도구와 수동 검토 사용
2. **링크 검증**: 모든 내부 및 외부 링크가 올바르게 작동하는지 확인
3. **형식 일관성**: 문서 전체에서 균일한 형식 보장
4. **기술적 정확성**: 모든 코드 예제와 지침 검증

#### 유지보수 절차

- **정기 업데이트**: 주기적인 콘텐츠 검토 및 업데이트 일정 수립
- **링크 확인**: 자동화된 링크 검증 구현
- **폐지 공지**: 오래된 콘텐츠를 명확하게 표시
- **버전 추적**: 중요한 업데이트에 대한 변경 로그 유지

이러한 마크다운 기초를 마스터하면 BaiDocs에서 명확하고 전문적인 기술 문서를 작성할 수 있는 기반을 제공합니다.
