# 경고문과 콜아웃

경고문은 중요한 정보, 경고 및 보조 콘텐츠를 시각적으로 구별되는 방식으로 강조할 수 있는 기능을 제공합니다. 이 섹션에서는 BaiDocs에서 사용 가능한 포괄적인 경고문 시스템을 다룹니다.

## 경고문 유형

### 정보성 경고문

#### Info
이해를 향상시키는 보조 정보에 사용:

```markdown
::: info
주요 워크플로에 중요하지는 않지만 메인 콘텐츠를 지원하는
추가 컨텍스트나 배경 정보입니다.
:::
```

#### Note
독자가 알아야 할 중요한 세부사항에 사용:

```markdown
::: note
현재 주제나 절차에 대한 명확한 설명이나 추가 컨텍스트를
제공하는 중요한 세부사항입니다.
:::
```

#### Tip
도움이 되는 제안과 모범 사례에 사용:

```markdown
::: tip
사용자 경험이나 효율성을 향상시킬 수 있는 도움이 되는 제안,
단축키 또는 모범 사례입니다.
:::
```

### 경고 경고문

#### Warning
잠재적으로 문제가 될 수 있는 상황에 사용:

```markdown
::: warning
문제를 피하기 위해 주의 깊은 관심이 필요한 잠재적 문제나
상황에 대한 정보입니다.
:::
```

#### Caution
신중한 고려가 필요한 상황에 사용:

```markdown
::: caution
의도하지 않은 결과를 방지하기 위해 신중한 고려나 특정 예방조치가
필요한 상황입니다.
:::
```

#### Danger
심각한 위험에 대한 중요 경고에 사용:

```markdown
::: danger
데이터 손실, 보안 취약점 또는 시스템 손상을 초래할 수 있는
행동에 대한 중요 경고입니다.
:::
```

### 특수 경고문

#### Example
실제 예시를 통해 개념을 보여주는 데 사용:

```markdown
::: example
개념이 실제 시나리오나 특정 사용 사례에서 어떻게 적용되는지
보여주는 실제적인 데모입니다.
:::
```

#### Quote
중요한 인용이나 인용문에 사용:

```markdown
::: quote
"문서화 콘텐츠를 지원하는 권위 있는 출처나 중요한 성명의
중요한 인용문입니다."
:::
```

## 경고문 내 콘텐츠 구조

### 단순한 콘텐츠

```markdown
::: warning
독자의 즉각적인 주의가 필요한 간단한 정보가 담긴
단일 문단 경고입니다.
:::
```

### 다중 문단 콘텐츠

```markdown
::: info 프로젝트 설정 요구사항

프로젝트 설정에는 여러 구성 요소가 올바르게 구성되어야 합니다.
각 구성 요소는 전체 아키텍처에서 특정 목적을 수행합니다.

구성 파일은 환경 설정을 반영하도록 업데이트되어야 합니다.
테스트 절차는 모든 구성 요소가 프로덕션 환경에 배포하기 전에
올바르게 작동하는지 확인해야 합니다.
:::
```

### 목록과 구조화된 콘텐츠

```markdown
::: tip 모범 사례 체크리스트

문서 배포 전에:

- **콘텐츠 검토**: 모든 콘텐츠가 정확하고 최신인지 확인
- **링크 검증**: 모든 내부 및 외부 링크 테스트
- **이미지 최적화**: 이미지가 적절히 압축되었는지 확인
- **접근성 확인**: 콘텐츠가 접근성 표준을 충족하는지 확인

추가 고려사항:
- 다양한 디바이스에서의 성능 테스트
- 크로스 브라우저 호환성 검증
- 검색 엔진 최적화 검토
:::
```

### 경고문 내 코드 예시

```markdown
::: example 데이터베이스 구성

환경 변수를 사용하여 데이터베이스 연결을 구성하십시오:

```env
DATABASE_URL=postgresql://user:password@localhost:5432/mydb
DATABASE_POOL_SIZE=20
DATABASE_TIMEOUT=5000
```

애플리케이션에서 연결을 구현하십시오:

```javascript
const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  max: process.env.DATABASE_POOL_SIZE,
  idleTimeoutMillis: process.env.DATABASE_TIMEOUT,
});
```
:::
```

시스템에 Node.js 버전 18 이상이 설치되어 있는지 확인하십시오.
:::

::: warning 백업 권장사항
설치 프로세스를 진행하기 전에 기존 구성의 백업을 생성하십시오.
:::

::: tip 설치 최적화
더 빠른 설치를 위해 npm 캐시를 사용하고 대체 패키지 매니저로
yarn 사용을 고려하십시오.
:::
```

### 중첩 콘텐츠 구조

```markdown
::: warning 중요 업데이트 절차

이 업데이트는 다음 단계의 신중한 실행이 필요합니다:

::: info 1단계: 준비
1. 실행 중인 모든 서비스 중지
2. 데이터베이스 백업 생성
3. 현재 구성 문서화

2. 구성 변경사항 적용
3. 스테이징 환경에서 테스트
:::

::: danger 최종 경고
성공적인 스테이징 환경 테스트와 검증 없이는
프로덕션 배포로 진행하지 마십시오.
:::

:::
```

## 접근성 고려사항

### 스크린 리더 호환성

- **의미론적 마크업**: 경고문은 적절한 ARIA 레이블과 의미론적 HTML을 사용
- **명확한 계층구조**: 경고문 유형이 보조 기술에 컨텍스트 제공
- **서술적 제목**: 사용자 정의 제목이 스크린 리더의 이해를 향상

### 시각적 접근성

- **색상 대비**: 모든 경고문 유형이 WCAG 색상 대비 요구사항을 충족
- **아이콘 사용**: 아이콘은 텍스트 정보를 대체하지 않고 보완
- **반응형 디자인**: 경고문이 다양한 화면 크기와 확대/축소 레벨에 적응

## 모범 사례

### 콘텐츠 가이드라인

#### 적절한 사용법

- **선택적 적용**: 진정으로 중요한 정보에만 경고문 사용
- **일관된 용어**: 유사한 경고문에서 일관된 언어 유지
- **명확한 컨텍스트**: 경고문이 콘텐츠 흐름을 방해하지 않고 향상시키도록 보장
- **논리적 배치**: 콘텐츠의 관련 지점에 경고문 배치

#### 콘텐츠 품질

```markdown
✅ 좋은 예시:
::: warning 데이터베이스 연결
애플리케이션을 시작하기 전에 데이터베이스 자격 증명이 구성되어 있는지
확인하십시오. 연결 실패는 적절한 시스템 초기화를 방해합니다.
:::

❌ 나쁜 예시:
::: warning
데이터베이스를 올바르게 설정하지 않으면 무언가 잘못될 수 있습니다.
:::
```

### 성능 고려사항

#### 효율적인 구현

- **최소 중첩**: 경고문 내 경고문의 깊은 중첩 제한
- **합리적인 길이**: 경고문 콘텐츠를 집중적이고 간결하게 유지
- **이미지 최적화**: 경고문 내에 포함된 이미지 최적화
- **코드 형식**: 경고문에서 구문 강조를 신중하게 사용

### 유지보수 및 업데이트

#### 콘텐츠 검토 프로세스

1. **정기 감사**: 정확성과 관련성을 위한 경고문 검토
2. **버전 업데이트**: 소프트웨어 버전 변경 시 경고문 업데이트
3. **사용자 피드백**: 경고문의 명확성과 유용성에 대한 사용자 피드백 모니터링
4. **일관성 검사**: 모든 문서에서 일관된 형식 보장

#### 품질 보증

- **링크 검증**: 경고문 내 모든 링크가 올바르게 작동하는지 확인
- **코드 테스트**: 경고문에 포함된 모든 코드 예시 테스트
- **상호 참조 검증**: 다른 섹션에 대한 참조가 정확한지 확인
- **접근성 테스트**: 보조 기술과 함께 경고문을 정기적으로 테스트

## 문서 워크플로와의 통합

### 편집 가이드라인

#### 검토 체크리스트

- [ ] 콘텐츠 중요성에 적합한 경고문 유형
- [ ] 사용자 정의 제목이 명확성과 컨텍스트 추가
- [ ] 콘텐츠가 간결하면서도 포괄적
- [ ] 형식이 가독성을 향상
- [ ] 링크와 참조가 작동함

#### 번역 고려사항

- **제목 번역**: 의미를 유지하면서 사용자 정의 제목 번역
- **문화적 적응**: 경고 해석에서 문화적 차이 고려
- **길이 조정**: 번역에서 텍스트 확장 또는 축약 고려
- **일관성 유지**: 언어 간 경고문 사용 일관성 보장

경고문의 효과적인 사용은 다양한 유형의 콘텐츠 중요성에 대한 명확한 시각적 계층구조를 제공하여 중요한 정보에 주의를 끌고 문서 사용성을 크게 향상시킵니다.
