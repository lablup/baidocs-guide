# 표와 목록

표와 목록을 통한 효과적인 데이터 구성은 문서 가독성과 정보 접근성을 향상시킵니다. 이 섹션에서는 BaiDocs의 포괄적인 표와 목록 형식 옵션을 다룹니다.

## 표 기초

### 기본 표 구조

```markdown
| 헤더 1 | 헤더 2 | 헤더 3 |
|--------|--------|--------|
| 행 1 열 1 | 행 1 열 2 | 행 1 열 3 |
| 행 2 열 1 | 행 2 열 2 | 행 2 열 3 |
| 행 3 열 1 | 행 3 열 2 | 행 3 열 3 |
```

### 열 정렬

```markdown
| 왼쪽 정렬 | 가운데 정렬 | 오른쪽 정렬 |
|:----------|:----------:|-----------:|
| 텍스트 | 텍스트 | 텍스트 |
| 더 긴 텍스트 | 더 긴 텍스트 | 더 긴 텍스트 |
| 짧음 | 짧음 | 짧음 |
```

## 특수 목록 형식

### 정의 목록

```markdown
**API 엔드포인트**
: 애플리케이션이 요청을 받고 응답을 보낼 수 있는 특정 URL입니다. RESTful API는 표준 HTTP 메서드(GET, POST, PUT, DELETE)를 사용하여 작업을 수행합니다.

**인증 토큰**
: 사용자 신원을 확인하고 보호된 리소스에 대한 액세스를 부여하는 데 사용되는 보안 자격 증명입니다. 토큰은 일반적으로 만료 시간과 특정 범위 제한이 있습니다.

**콘텐츠 관리 시스템(CMS)**
: 사용자가 기술적 프로그래밍 지식 없이도 디지털 콘텐츠를 생성, 관리, 게시할 수 있게 해주는 소프트웨어 애플리케이션입니다.

**마크다운**
: 헤더, 강조, 링크 및 기타 요소에 대한 간단한 문법 규칙을 사용하여 일반 텍스트 문서를 형식화할 수 있는 경량 마크업 언어입니다.

**버전 관리**
: 시간이 지남에 따라 파일의 변경 사항을 추적하고 관리하는 시스템으로, 협업, 변경 이력, 이전 버전으로 되돌릴 수 있는 능력을 가능하게 합니다.
```

## 표와 목록 모범 사례

### 접근성 고려사항

#### 표 접근성

```markdown
| 기능 | 설명 | 스크린 리더 지원 | 키보드 네비게이션 |
|------|------|-----------------|-------------------|
| 헤더 | 명확한 열 및 행 헤더 | 적절히 알림 | 탭 네비게이션 |
| 콘텐츠 | 서술적인 셀 콘텐츠 | 의미 있는 컨텍스트 | 포커스 표시기 |
| 구조 | 논리적 구성 | 표 구조 알림 | 건너뛰기 네비게이션 |
```

#### 목록 접근성

- **의미론적 마크업**: 스크린 리더 호환성을 위한 적절한 목록 요소 사용
- **명확한 계층구조**: 논리적인 중첩 레벨과 구조 유지
- **서술적 콘텐츠**: 의미 있는 목록 항목 콘텐츠 제공
- **네비게이션 보조**: 긴 목록을 위한 건너뛰기 링크 포함

### 모바일 최적화

#### 반응형 표

```markdown
| 속성 | 값 | 설명 | 브라우저 지원 |
|------|-----|------|-------------|
| `overflow-x` | `auto` | 수평 스크롤 활성화 | 모든 최신 브라우저 |
| `white-space` | `nowrap` | 텍스트 줄바꿈 방지 | 모든 브라우저 |
| `min-width` | `300px` | 최소 표 너비 설정 | 모든 브라우저 |
```

모바일 디바이스의 경우 표는 가독성을 유지하기 위해 자동으로 수평 스크롤을 활성화합니다.

#### 모바일 친화적 목록

- **컴팩트한 형식**: 모바일 화면을 위한 간결한 목록 항목 사용
- **터치 최적화**: 터치 상호 작용을 위한 적절한 간격 보장
- **가독 가능한 타이포그래피**: 모바일 보기에 적합한 글꼴 크기 유지

### 성능 고려사항

#### 대용량 표 최적화

- **콘텐츠 페이지네이션**: 대용량 표를 여러 페이지로 분할
- **점진적 로딩**: 필요에 따라 표 콘텐츠 로딩
- **검색 및 필터링**: 특정 정보 찾기를 위한 도구 제공
- **내보내기 옵션**: 오프라인 분석을 위한 데이터 내보내기 허용

#### 목록 성능

- **적절한 길이**: 목록을 집중적이고 관리 가능하게 유지
- **지연 로딩**: 긴 목록을 점진적으로 로딩
- **계층적 접기**: 중첩된 목록 섹션의 접기 허용

### 편집 가이드라인

#### 콘텐츠 품질 표준

```markdown
✅ 효과적인 표 설계:
- 명확하고 서술적인 헤더
- 일관된 데이터 형식
- 논리적인 행과 열 구성
- 데이터 유형에 적합한 정렬

❌ 잘못된 표 설계:
- 모호하거나 누락된 헤더
- 일관성 없는 형식
- 비논리적인 구성
- 정렬되지 않은 콘텐츠
```

#### 목록 콘텐츠 가이드라인

- **병렬 구조**: 일관된 문법 구조 유지
- **논리적 순서**: 중요도, 시간순 또는 카테고리별로 항목 구성
- **적절한 깊이**: 명확성을 위해 중첩 레벨 제한
- **명확한 관계**: 목록 계층구조가 콘텐츠 관계를 반영하도록 보장

잘 구조화된 표와 목록은 복잡한 정보를 소화하기 쉽고 스캔 가능한 형식으로 구성하여 사용자 이해와 작업 완료를 향상시켜 문서 사용성을 크게 개선합니다.
