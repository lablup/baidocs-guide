## Importing from Sphinx

### Preparation

Ensure your Sphinx project has:
- `conf.py` configuration file
- Structured documentation with `.rst` files
- Built HTML version (for reference)

### Import Configuration

![Sphinx Import Configuration](../_assets/sphinx-import-config.png)

#### Source Configuration

- **Repository URL**: `https://github.com/your-org/sphinx-docs.git`
- **Branch**: Source branch (usually `main` or `master`)
- **Format**: Select "Sphinx"

#### Target Configuration

- **Target Name**: New BaiDocs book name (e.g., `api-documentation`)
- **Display Title**: Human-readable title
- **Emoji**: Visual identifier for the book

#### Conversion Options

- **Language Mapping**: Map Sphinx languages to BaiDocs
  ```
  Sphinx: en â†’ BaiDocs: en
  Sphinx: es â†’ BaiDocs: es
  ```

- **Content Processing**:
  - âœ… Convert RST to MDX
  - âœ… Extract navigation from toctree
  - âœ… Migrate images and assets
  - âœ… Convert code blocks with syntax highlighting

### Conversion Results

**Before (Sphinx)**:

## Importing from MkDocs

### Preparation

Verify your MkDocs project includes:
- `mkdocs.yml` configuration file
- Markdown files in `docs/` directory
- Properly structured navigation

### Import Configuration

#### Source Analysis

BaiDocs analyzes your MkDocs configuration:

```yaml
# mkdocs.yml (original)
site_name: User Guide
site_description: Complete user documentation
nav:
  - Home: index.md
  - Getting Started:
    - Installation: getting-started/install.md
    - Quickstart: getting-started/quickstart.md
  - Advanced: advanced.md
```

#### Conversion Process

- **Configuration Migration**: `mkdocs.yml` â†’ `book.config.yaml`
- **Content Conversion**: Markdown â†’ MDX (enhanced features)
- **Theme Translation**: MkDocs themes â†’ BaiDocs styling
- **Plugin Conversion**: Common plugins â†’ BaiDocs equivalents

### Conversion Results

**After Conversion**:
```yaml

id: user-guide
title: User Guide
description: Complete user documentation
navigation:
  en:
    - title: "Home"
      path: "index.mdx"
    - title: "Getting Started"
      items:
        - title: "Installation"
          path: "getting-started/install.mdx"
        - title: "Quickstart"
          path: "getting-started/quickstart.mdx"
    - title: "Advanced"
      path: "advanced.mdx"
```

## Troubleshooting Import Issues

### Common Import Problems

#### "Unsupported source format"
- Verify the repository contains documentation in a supported format
- Check for required configuration files (conf.py, mkdocs.yml, etc.)
- Consider using generic Markdown import

#### "Navigation extraction failed"
- Source navigation may be complex or non-standard
- Manual navigation editing may be required after import
- Check for malformed navigation files

#### "Asset conversion errors"
- Some assets may have unsupported formats
- Review asset paths in source documentation
- Manual asset migration may be needed

### Recovery and Manual Fixes

#### Post-import Manual Editing

1. **Review Generated Navigation**: Adjust structure as needed
2. **Fix Broken Links**: Update references that didn't convert correctly
3. **Enhance Content**: Add BaiDocs-specific features
4. **Test All Pages**: Verify content renders correctly

#### Re-import with Different Settings

```bash
# Remove failed import
rm -rf content/failed-import

# Try with different configuration:
# - Different target structure
# - Modified source branch
# - Alternative format detection
```

## Best Practices

### Before Import

- **Backup Original**: Ensure original documentation is safely backed up
- **Clean Source**: Remove unnecessary files and artifacts
- **Document Process**: Note custom configurations or special requirements

### During Import

- **Monitor Progress**: Watch import logs for warnings or errors
- **Review Configurations**: Verify target settings are correct
- **Test Incrementally**: Import small sections first for complex projects

### After Import

- **Content Review**: Manually review converted content
- **Link Verification**: Check all internal and external links
- **Navigation Testing**: Ensure navigation works correctly
- **Team Review**: Have team members review converted content
