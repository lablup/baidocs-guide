# Overview

BaiDocs is a modern documentation framework that separates content creation from content presentation, providing a flexible and powerful solution for teams of all sizes.

## Architecture Overview

BaiDocs follows a **dual-application architecture** that separates concerns between editing and viewing:

### ğŸ“ **Editor Application** (`/apps/editor`)

The Editor is a Next.js application that provides:

- **Content Management**: Create, edit, and organize documentation
- **Git Integration**: Connect to remote repositories, manage branches
- **Import Tools**: Convert from Sphinx, MkDocs, GitBook formats
- **Live Preview**: Real-time preview of content changes
- **Multi-language Support**: Manage translations and language-specific content

**Primary Use Case**: Content creators and documentation maintainers use the Editor locally to create and manage documentation.

### ğŸ‘€ **Viewer Application** (`/apps/viewer`)

The Viewer is a Next.js application optimized for public consumption:

- **Fast Rendering**: Optimized for performance and SEO
- **Responsive Design**: Mobile-first, accessible interface
- **Search Integration**: Full-text search across all content
- **Version Support**: Browse different versions/branches
- **Deployment Ready**: Easy deployment to Vercel, Netlify, etc.

**Primary Use Case**: End users and documentation consumers access the Viewer for reading documentation.

### ğŸ“„ **Standard Markdown**

### ğŸ§¹ **Clean Import Process**

When you import documentation:

1. **Analysis**: BaiDocs analyzes the source structure
2. **Conversion**: Content is converted to MDX format
3. **Clean Repository**: Creates a new, clean Git repository
4. **Structure Optimization**: Organizes content in BaiDocs format
5. **Asset Migration**: Images and files are properly organized

The result is a clean, optimized BaiDocs repository without the baggage of the original tooling.
