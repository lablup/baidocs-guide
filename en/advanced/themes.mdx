# Themes & Customization

BaiDocs provides a flexible theming system that allows you to customize the visual appearance of your documentation while maintaining consistency and accessibility across all platforms.

## Current Theme System

### Default Theme

BaiDocs ships with a carefully designed default theme that provides:

- **Modern Design**: Clean, professional layout with excellent readability
- **Responsive Layout**: Optimized for all screen sizes from mobile to desktop
- **Dark Mode Support**: Automatic theme switching based on user preference
- **Accessibility**: WCAG 2.1 AA compliance with proper color contrast
- **Typography**: Optimized font stacks for excellent reading experience

### Theme Architecture

The BaiDocs theme system is built on:

```
Theme Structure:
├── Layout Components     # Navigation, headers, footers
├── Content Components    # Typography, code blocks, tables
├── Interactive Elements  # Buttons, forms, search
├── Color System         # Light/dark mode variables
└── Typography Scale     # Responsive font sizing
```

## Customization Options

### CSS Variables

Override default styling using CSS custom properties:

```css
/* Custom theme variables */
:root {
  /* Primary colors */
  --color-primary: #2563eb;
  --color-primary-hover: #1d4ed8;

  /* Background colors */
  --color-background: #ffffff;
  --color-background-secondary: #f8fafc;

  /* Text colors */
  --color-text-primary: #1f2937;
  --color-text-secondary: #6b7280;

  /* Border and accent colors */
  --color-border: #e5e7eb;
  --color-accent: #10b981;
}

/* Dark mode overrides */
[data-theme="dark"] {
  --color-background: #111827;
  --color-background-secondary: #1f2937;
  --color-text-primary: #f9fafb;
  --color-text-secondary: #d1d5db;
  --color-border: #374151;
}
```

```css
/* Typography overrides */

  --text-xl: 1.25rem;
  --text-2xl: 1.5rem;

  /* Line heights */
  --leading-none: 1;
  --leading-tight: 1.25;
  --leading-normal: 1.5;
  --leading-relaxed: 1.625;
}
```

## Custom Styling Implementation

### Global Styles

Add custom CSS to your documentation:

1. **Create custom stylesheet**:

```css
/* styles/custom.css */
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap');

:root {
  --brand-primary: #6366f1;
  --brand-secondary: #8b5cf6;
  --brand-accent: #06b6d4;
}

/* Custom brand styling */
.header-logo {
  color: var(--brand-primary);
  font-weight: 600;
}

.navigation-active {
  background-color: var(--brand-primary);
  color: white;
  border-radius: 0.375rem;
}

/* Enhanced code blocks */
pre[class*="language-"] {
  background: linear-gradient(135deg, #1e293b 0%, #334155 100%) !important;
  border: 1px solid #475569;
  box-shadow: 0 4px 6px -1px rgba(0, 0, 0, 0.1);
}

/* Custom admonition styles */
.admonition-tip {
  background: linear-gradient(135deg, #ecfdf5 0%, #d1fae5 100%);
  border-left: 4px solid #10b981;
}

.admonition-warning {
  background: linear-gradient(135deg, #fffbeb 0%, #fef3c7 100%);
  border-left: 4px solid #f59e0b;
}
```

2. **Include in your documentation**:

```mdx
<link rel="stylesheet" href="/styles/custom.css" />

# Your Documentation

Content with custom styling...
```

## Best Practices

### Performance Considerations

**Optimize Custom CSS**:
```css
/* Good: Use CSS variables */
.custom-element {
  color: var(--brand-primary);
  background: var(--surface-color);
}

/* Avoid: Hardcoded values */
.custom-element {
  color: #6366f1;
  background: #ffffff;
}
```

**Minimize CSS Bundle Size**:
```css
/* Use efficient selectors */
.navigation-item { /* Good */ }
div.nav ul li a { /* Avoid - too specific */ }

/* Leverage existing utilities */
.text-brand { color: var(--brand-primary); }
.bg-surface { background: var(--surface-color); }
```

### Accessibility Guidelines

**Maintain Color Contrast**:
```css
/* Ensure WCAG AA compliance */
:root {
  --text-on-primary: #ffffff; /* 4.5:1 ratio minimum */
  --text-on-surface: #1f2937; /* High contrast */
}

/* Test with accessibility tools */
.custom-link {
  color: var(--link-color);
  text-decoration: underline;
  text-underline-offset: 0.125rem;
}

.custom-link:focus {
  outline: 2px solid var(--focus-ring);
  outline-offset: 2px;
}
```

**Support Dark Mode**:
```css
/* Automatic theme adaptation */
@media (prefers-color-scheme: dark) {
  :root {
    --surface-color: #111827;
    --text-color: #f9fafb;
    --border-color: #374151;
  }
}

/* Manual theme toggle support */
[data-theme="dark"] {
  --surface-color: #111827;
  --text-color: #f9fafb;
}
```

### Responsive Design

**Mobile-First Approach**:
```css
/* Base styles for mobile */
.custom-grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 1rem;
}

/* Tablet and up */
@media (min-width: 768px) {
  .custom-grid {
    grid-template-columns: repeat(2, 1fr);
    gap: 1.5rem;
  }
}

/* Desktop and up */
@media (min-width: 1024px) {
  .custom-grid {
    grid-template-columns: repeat(3, 1fr);
    gap: 2rem;
  }
}
```
