# Multi-language Support

BaiDocs provides comprehensive multi-language support, allowing you to create documentation in multiple languages with automatic language detection, translation workflows, and localized user experiences.

## Overview

### Multi-language Features

- **Multiple Language Support**: Create content in any number of languages
- **Language-Specific Navigation**: Independent navigation structures per language
- **Automatic Language Detection**: Detect user's preferred language
- **Language Switching**: Seamless switching between language versions
- **RTL Support**: Right-to-left language support (Arabic, Hebrew, etc.)
- **Localized UI**: Translate interface elements and messages
- **Search Integration**: Language-aware search functionality

### Supported Languages

BaiDocs supports all languages with proper locale codes:

- **Western Languages**: English (en), Spanish (es), French (fr), German (de), Italian (it)
- **Asian Languages**: Korean (ko), Japanese (ja), Chinese Simplified (zh-cn), Chinese Traditional (zh-tw)
- **RTL Languages**: Arabic (ar), Hebrew (he), Persian (fa)
- **Other Languages**: Russian (ru), Portuguese (pt), Dutch (nl), and many more

#### Translation Validation

Automated translation validation:

```typescript
// scripts/validate-translations.ts
interface TranslationValidation {
  missingTranslations: string[];
  outdatedTranslations: string[];
  structureMismatches: string[];
  brokenLinks: string[];
}

async function validateTranslations(
  baseLanguage: string = 'en',
  targetLanguages: string[]
): Promise<Record<string, TranslationValidation>> {
  const results: Record<string, TranslationValidation> = {};

  for (const lang of targetLanguages) {
    results[lang] = {
      missingTranslations: [],
      outdatedTranslations: [],
      structureMismatches: [],
      brokenLinks: []
    };

    // Check for missing files
    const baseFiles = await getLanguageFiles(baseLanguage);
    const langFiles = await getLanguageFiles(lang);

    for (const file of baseFiles) {
      const langFile = file.replace(`/${baseLanguage}/`, `/${lang}/`);
      if (!langFiles.includes(langFile)) {
        results[lang].missingTranslations.push(langFile);
      }
    }

    // Check for outdated translations
    for (const file of langFiles) {
      const baseFile = file.replace(`/${lang}/`, `/${baseLanguage}/`);
      if (await isFileNewer(baseFile, file)) {
        results[lang].outdatedTranslations.push(file);
      }
    }

    // Validate navigation structure
    const baseNav = await getNavigationStructure(baseLanguage);
    const langNav = await getNavigationStructure(lang);

    if (!navigationStructuresMatch(baseNav, langNav)) {
      results[lang].structureMismatches.push('navigation structure');
    }

    // Check for broken internal links
    const brokenLinks = await validateInternalLinks(lang);
    results[lang].brokenLinks = brokenLinks;
  }

  return results;
}
```

## Language Detection and Switching

### Automatic Language Detection

Detect user's preferred language:

```typescript
// Language detection logic
function detectUserLanguage(
  availableLanguages: string[],
  defaultLanguage: string = 'en'
): string {
  // 1. URL language parameter
  const urlParams = new URLSearchParams(window.location.search);
  const urlLang = urlParams.get('lang');
  if (urlLang && availableLanguages.includes(urlLang)) {
    return urlLang;
  }

  // 2. Stored user preference
  const storedLang = localStorage.getItem('preferred-language');
  if (storedLang && availableLanguages.includes(storedLang)) {
    return storedLang;
  }

  // 3. Browser language
  const browserLangs = navigator.languages || [navigator.language];
  for (const browserLang of browserLangs) {
    // Exact match
    if (availableLanguages.includes(browserLang)) {
      return browserLang;
    }

    // Language code match (en-US -> en)
    const langCode = browserLang.split('-')[0];
    if (availableLanguages.includes(langCode)) {
      return langCode;
    }
  }

  // 4. Default language
  return defaultLanguage;
}
```
